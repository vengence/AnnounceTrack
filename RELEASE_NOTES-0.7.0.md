# 监控助手 0.7.0

这一版在已验证的公告监控核心上补齐 Windows 11 交付、低资源运行和安全边界。

## 新增

- Windows 11 x64 应用与 NSIS 安装目标；GitHub Actions 在 Windows 原生环境自动构建。
- 可在 Apple Silicon macOS 直接生成 Windows x64 便携 ZIP 的独立构建命令。
- 页面中显示当前版本号。
- README 产品截图、跨平台安装说明和发布工作流。
- 当公告列表仍正常、但本轮所有公告详情都无法按配置提取时，累计失败并发送“采集配置可能失效”告警。

## 优化

- 网络响应单项限制 1.5 MB、单次总缓冲限制 24 MB，最多保留 96 个响应；日常监控只取目标响应。
- 隐藏浏览器阻止图片、字体、音视频资源，启用后台节流，并在采集后立即释放请求映射和窗口。
- UI 和任务运行热路径不再复制完整正文历史；后台进度不再反复刷新整个界面状态。
- 数据文件使用紧凑 JSON，历史版本每条公告保留最近 20 个；30 天后移除原始响应和 HTML。
- DeepSeek、正文、DOM、日志和公开状态均设定明确上限。

## 修复与安全

- Windows 托盘图标使用非模板彩色图标，macOS 保持模板图标。
- 邮件运行依赖 `nodemailer` 随安装包打入 ASAR。
- 公告正文进入历史界面和邮件前清理脚本、嵌入对象、事件属性、危险协议和不可信样式。
- 修复快速跳转返回 `ERR_ABORTED (-3)` 时误判为页面打开失败的问题。

升级不会主动删除现有任务。版本保留策略会在下次数据写入时自动应用。
