# 监控助手 0.7.1

这一版集中修复多个真实公告站点的采集兼容性，并补齐任务规模扩大后的组织、诊断和资源控制能力。

## 采集修复

- 快手与抖店：扩展重复 DOM 卡片识别，不再局限于链接、`li` 或 ARIA 列表项；网络候选必须与用户样例有实际重合，避免无关数组被选中。
- 苏宁服务市场：识别隐藏公告 ID 与 `onclick` 交互，可按 ID 或标题点击列表项后提取动态加载的正文；缺少可执行详情规则时会明确记为配置失败。
- 淘宝：移除超大剪贴板属性，避免截断产生无效 HTML；历史界面检测到 HTML 正文覆盖不足时自动回退到完整纯文本。
- DeepSeek 返回百分数字符串时可正确解析置信度；低置信度不可执行方案不会再被保存为 0% 方案。

## 新功能

- 任务创建后可重新分析获取方案，保存后的第一次检查静默重建基线，不触发变化告警。
- 支持任务分组、侧栏滚动、最新 N 条公告限制、固定每日检查时间和固定内容排除。
- 左下角新增任务队列和日志入口；队列弹窗显示当前执行项与等待顺序。
- “需要处理”可打开异常详情并跳转到对应任务。
- 设置页新增本地存储占用、DeepSeek 调用/token 统计，以及可编辑的新公告摘要和内容对比提示词。
- 页面监控通知使用“页面”文案，与公告监控明确区分。
- 版本号可直接打开 GitHub 仓库。
- 方案生成或验证失败时可一键复制完整、脱敏的诊断材料。

升级保留现有任务与历史。旧任务可继续运行；需要使用分组、最新 N 条或重新分析方案时，在任务设置中补充即可。
